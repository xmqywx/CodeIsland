配对你的 iPhone

把 Mac 上的 Mio Island 和 iPhone 上的 CodeLight 连到一起, Claude Code 的状态、消息、审批请求会实时同步到你的灵动岛和锁屏 — 人不在电脑前也能掌控全局。

准备时间 · 约 5 分钟 Mio Island · v2.1.9+ iPhone · iOS 16.1+
0

下载两端 App

  • Mac · Mio Island (开源免费)
    brew install --cask xmqywx/codeisland/codeisland 或者 下载最新 DMG
  • iPhone · CodeLight
    App Store 搜索 "Code Light" 下载 (需要 iOS 16.1+, 支持 Dynamic Island 的机型体验最佳)

在 Mac 上打开 "Pair iPhone"

启动 Mio Island 后, 鼠标移到刘海上, 菜单展开 → 点 Pair iPhone。

或者: 系统设置 → CodeLight 标签 → 右上 "配对新手机" 按钮。

[截图 02] Mio Island 刘海菜单 / 系统设置 CodeLight 面板的 "Pair iPhone" 入口 截图建议: 刘海展开态, "Pair iPhone" 那一行被鼠标悬停高亮
2

填入服务器地址

首次打开会看到一个服务器机架图标 + 输入框。直接填官方服务器地址:

https://code.7ove.online

点 Save and Connect, 等 1-2 秒连接。

[截图 03] 未配置状态下的服务器 URL 输入面板, 已填入 https://code.7ove.online 截图建议: 柠檬绿卡片, 中间服务器机架图标 + 输入框里已有 code.7ove.online + "Save and Connect" 按钮高亮
💡 为什么走官方服务器 CodeLight 所有功能 (Dynamic Island 实时推送、多端同步、远程审批、会话转发) 都走服务器中转, 协议端到端加密, 服务器只做零知识转发、不落盘、不记 IP 日志, 看不到消息明文。官方服务器已经调优好 APNs 推送、并发控制、试用/订阅状态管理, 直接用最省事。地址仅保存在你本机 UserDefaults, 不上传任何地方。
3

获取二维码和 6 位配对码

连接成功后, 面板会切到配对就绪状态:

  • 中间是 二维码 (约 160×160), iPhone 直接扫
  • 下面是 6 位大字配对码 (比如 X7K2M9), 也可以手动输入
  • 底部两个胶囊: 服务器地址 (点击可改) + 当前 Mac 设备名
[截图 04] 配对就绪态: 二维码 + 6 位配对码同屏显示 截图建议: 同一张图同时显示 QR code + 下面的大字码 X7K2M9, 加两个胶囊 (服务器 pill + 设备名 pill)
💡 配对码永久有效 同一个码可以配多台 iPhone (工作 + 私人机), 不过期、不自动刷新。只有换服务器才重新生成。
4

在 iPhone 上打开 CodeLight 并配对

下载 CodeLight 后首次启动, 会看到配对引导:

  • 方式 A · 扫码: 允许相机权限 → 对准 Mac 上的二维码 → 服务器地址 + 配对码自动填好
  • 方式 B · 手输: 服务器 URL 填 https://code.7ove.online, 配对码填上面那个 6 位 X7K2M9
  • 点 "配对" 按钮, 等 1-2 秒
[截图 05] iPhone CodeLight 首屏 · 配对输入界面 截图建议: iPhone 竖屏, 输入框里显示 code.7ove.online + 配对码输入光标

6

开启 Hooks (实时状态同步)

配对只是建了通道, 要让 Claude Code 状态真正实时流过去, 需要 Mio Island 监听它。

系统设置 → 通用 → 找 钩子 开关, 确认打开 (绿色)。

首次启动自动安装, 没开手动打开即可。

[截图 09] 系统设置 · 通用 · "钩子" 开关 ON 截图建议: 设置窗口截图, 聚焦 General tab 里 "钩子" 那个 toggle 亮着绿色
💡 Hook 做什么 安装一个 Python 脚本到 ~/.claude/hooks/codeisland-state.py, 配到 Claude Code 的 ~/.claude/settings.json. 每次 Claude 状态变化 (开始 / 跑工具 / 完成 / 审批) 通过 Unix socket 推给 Mio Island, 再通过服务器转发到 iPhone. 纯本地 IPC, 不走网络。

跑一次真实会话验证

  1. Mac 上打开终端, 在项目目录执行 claude
  2. 问一个需要调工具的问题 (比如 "看下当前目录结构")
  3. 观察三处: Mac 刘海状态点 + 文字 / iPhone Dynamic Island 或锁屏 Live Activity
  4. 几百毫秒内三端同步显示 "Working..." / 工具名 / 完成

进阶操作

换服务器: 点 Pair 面板下方的服务器胶囊 (带铅笔图标) → 输入新 URL → Save. 之前配过的 iPhone 要重新扫码。

常见问题

服务器 URL 填什么?

固定填官方地址 https://code.7ove.online。CodeLight 的 APNs 推送、并发控制、试用/订阅状态、兑换码校验都在这个服务器上。

CodeLight 怎么免费用?

两条路:

  • App Store 下载后 3 天免费试用 — 无需信用卡, 首次连接自动开始
  • 发内容换 30 天兑换码 — 公开平台发一条 (体验 / 教程 / 吐槽都行), 截图私信开发者, 审核通过就发码. 每 30 天发一条, 持续免费

彻底长期用, 直接 $13.99 买断, 三台设备通用, 支持 Apple ID 恢复购买。

服务器能看到我的代码 / 消息吗?

不能。协议端到端加密 (Ed25519 签名 + 对称加密), 服务器只做消息转发, 不落盘、不记日志、不记 IP, 看不到消息内容。

服务器数据库只存最小化运行信息: 设备配对码、试用/付费状态、最后连接时间、APNs 推送 token, 从不保存消息内容、代码文件或任何你在 Claude Code 里输入的内容。

服务器 URL 保存不了 / 一直转圈

检查:

  • URL 必须 https:// 或 http:// 开头
  • 网络能到达 — 浏览器打开 https://code.7ove.online 能有响应
  • 防火墙 / VPN / 代理是否挡了 WebSocket (CodeLight 走 Socket.io)
二维码出不来, 只显示转圈

说明服务器连接没成功。通常:

  • 服务器暂时不可达 — 等几分钟或浏览器试一下
  • DNS / 代理问题
  • Mio Island 版本太老 — 升级到 v2.1.9+
iPhone 配对成功了, 但状态不同步

99% 是 hook 没装:

  • 系统设置 → 通用 → 钩子开关打开
  • 文件检查: ls ~/.claude/hooks/codeisland-state.py
  • Claude Code 配置: ~/.claude/settings.json 里 hooks 字段应该多个事件指向 codeisland-state.py
  • 重启 Claude Code 会话 (老会话的 hook 是启动时加载的)
一台 Mac 能配几台 iPhone? 一台 iPhone 能配几台 Mac?

多对多:

  • 同一个 6 位配对码可以被多台 iPhone 扫 (办公 + 私人机)
  • 同一台 iPhone 可以连多台 Mac, 在 CodeLight 里切换
  • 官方服务器同时在线设备数: 付费账户最多 3 台并发
试用期过了, 怎么办?

两条路:

  • App 内 $13.99 买断 — 永久解锁, 同一 Apple ID 最多 3 台设备并发, 支持恢复购买
  • 参与 "发一条, 免费用" 活动换 30 天兑换码 — 公开平台发一条关于 MioIsland/Code Light 的内容, 截图私信开发者, 审核通过即发码. 持续发一直免费
想完全停用 Mio Island 的钩子

系统设置 → 通用 → 钩子关掉. 它会:

  • 从 ~/.claude/settings.json 删除 Mio Island 的 hook 入口 (其他工具的 hook 不动)
  • 删除 ~/.claude/hooks/codeisland-state.py
  • 刘海仍能用, 状态同步停止 (显示最后一次快照)
