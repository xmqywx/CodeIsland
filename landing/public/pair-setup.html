配对你的 iPhone

用 Mio Island (Mac) 和 CodeLight (iPhone) 打通一条加密通道, Claude Code 的状态 / 消息 / 审批请求都能实时同步到你的手机上, 哪怕人不在电脑前。

准备时间 · 约 5 分钟 需要 · Mac + iPhone + 一个中继服务器 Mio Island · v2.1.9+
0

你需要准备什么

  • Mac 端: 安装 Mio Island (brew install --cask xmqywx/codeisland/codeisland 或 下载 DMG)
  • iPhone 端: App Store 下载 CodeLight
  • 中继服务器: 一台能跑 MioServer 的机器 (自建) 或公共服务器的 URL
💡 为什么需要服务器 服务器是 Mac 和 iPhone 之间的零知识转发器, 消息端到端加密, 服务器看不到明文。你可以用 Docker 自己部署一个 (MioServer 仓库), 或者用朋友/团队共享的实例。
1

在 Mac 上打开 "Pair iPhone"

启动 Mio Island 后, 鼠标移到屏幕顶部刘海区域, 刘海展开 → 点击齿轮 ⚙️ 菜单 → 选 Pair iPhone。

或者: 系统设置 → CodeLight 标签页 → 点右上角 "配对新手机" 按钮。

[截图 01] Mio Island 刘海菜单 / 系统设置 CodeLight 面板的 "Pair iPhone" 入口 截图建议: 显示刘海展开态, 菜单里 "Pair iPhone" 那一行被悬停高亮
2

配置中继服务器地址

首次打开会看到 服务器机架图标 + 一个输入框 — 因为还没配过服务器, 二维码暂时生成不出来。

  • 在输入框里粘贴你的服务器 URL, 例如 https://relay.example.com
  • 必须是 https:// 或 http:// 开头, 并且带合法 host
  • 点 Save and Connect → 会显示转圈 1-2 秒表示正在连
[截图 02] "CodeLight Server" 未配置状态下的服务器 URL 输入面板 截图建议: 柠檬绿卡片, 中间是服务器机架图标 + 输入框 + "Save and Connect" 按钮
⚠️ 地址保存在哪里 仅保存在本机 (UserDefaults), 不会上传任何地方。换电脑要重填。
3

查看二维码 + 6 位配对码

连接成功后, 面板会自动切到 配对就绪 状态:

  • 顶部 二维码 (白底黑像素, 约 160×160) — iPhone 直接扫
  • 下面一个 6 位大字配对码 (比如 X7K2M9) — 手动输入备用
  • 最下面两个胶囊: 服务器地址 (可点击修改) + 当前 Mac 的设备名
[截图 03] Mac 上 Pair iPhone 面板的二维码 + 6 位配对码态 截图建议: 同一张图既显示 QR code 又显示下面的 X7K2M9 大字码, 加两个胶囊 (服务器 pill + 设备名 pill)
💡 配对码永久有效 同一个 6 位码可以配多台 iPhone, 不过期、不自动刷新 (除非你更换服务器)。
4

在 iPhone 上打开 CodeLight 并配对

从 App Store 下载安装 CodeLight, 启动后进入配对流程:

  • 方式 A · 扫码: 允许相机权限 → 对准 Mac 上的二维码 → 自动识别并填入服务器 URL + 配对码
  • 方式 B · 手输: 输入服务器 URL (和 Mac 填的一样) + 6 位配对码 X7K2M9
  • 点 "配对" 按钮, 等待约 1-2 秒
[截图 04] iPhone CodeLight 首屏 · 配对输入界面 (服务器 + 配对码) 截图建议: 两张小图并排: 左边 App Store 下载页面, 右边 CodeLight 配对输入界面
[截图 05] iPhone 扫描 Mac 二维码的瞬间 (相机取景框对准 QR code) 截图建议: 用手机拍 Mac 屏幕上 QR code 的画面, 展示 "扫码即可" 的体感
5

验证连接是否成功

配对后立刻检查两端状态, 两边都绿灯才算通:

  • Mac 端 — 面板底部多出 "Linked Devices" 一栏, 列出了刚配的 iPhone 名字 (比如 "Carey's iPhone")
  • Mac 刘海菜单 — "Pair iPhone" 那一行右侧出现柠檬绿点 + "Online" 字样
  • iPhone 端 — "Macs" 列表里多出这台 Mac, 可以点进去看会话
[截图 06] Mac 面板上 "Linked Devices" 区块显示 iPhone 已连接 (每行有 📱 图标 + 名字 + 可关闭的 ✕) 截图建议: 展开后面板下半部分, 能看到 "Linked Devices 1" 和 iPhone 条目
[截图 07] iPhone CodeLight Macs 列表显示 Mac 已配对 截图建议: iPhone 正竖屏截图, 主界面 Macs 列表第一条是刚配的 Mac 名字 + 在线小绿点
6

开启 Hooks (实时状态同步)

配对只是建了通道, 要让 Claude Code 的状态真正实时流到手机, 还得让 Mio Island 监听到它。

打开 Mio Island → 系统设置 → 通用 → 找到 钩子 那一行, 确认开关是打开状态 (绿色)。

首次启动会自动安装, 没开就手动打开即可。

[截图 08] 系统设置 · 通用 · "钩子" 开关处于 ON 状态 截图建议: 系统设置窗口截图, 聚焦在 General tab 里 "钩子" 那个 toggle 亮着绿色
💡 Hook 做什么 安装一个 Python 脚本到 ~/.claude/hooks/codeisland-state.py, 配到 Claude Code 的 ~/.claude/settings.json 里。每次 Claude 运行状态变化 (开始思考 / 跑工具 / 完成 / 要审批) 都通过 Unix socket 推给 Mio Island, 再转发到 iPhone。纯本地 IPC, 不走网络。
7

端到端测试

最后跑一次真实会话验证整条链路:

  1. Mac 上打开任意终端, 在项目目录里执行 claude
  2. 问一个需要调工具的问题, 比如 "看下当前目录文件"
  3. 观察三处: 刘海状态点 / 刘海文字 / iPhone Dynamic Island 或锁屏 Live Activity
  4. 三端应该都在几百毫秒内显示同步的 "Working..." / 工具名 / 完成
[截图 09] Mac 刘海 + iPhone Dynamic Island 同步显示 "Working · Read" 的瞬间 截图建议: 两张同一时刻的图合并: 左 Mac 刘海截图, 右 iPhone 锁屏 Live Activity, 都显示同一个会话状态
+

进阶操作

换服务器: 点 Pair 面板下方的服务器胶囊 (带铅笔图标) → 输入新 URL → Save。原来已配对的 iPhone 需要重新扫码。

解绑单台 iPhone: "Linked Devices" 里每一条最右边的 ✕ 按钮。

完全重置: 系统设置 → CodeLight → 删除所有 linked devices, 再清空服务器 URL。

[截图 10] 点击服务器胶囊进入 "Change Server" 编辑态 截图建议: 服务器胶囊被点击后, 面板切换到输入 URL 的编辑模式, 下面有 Cancel + Save 两个按钮

常见问题

服务器 URL 保存不了 / 一直转圈

检查:

  • URL 必须 https:// 或 http:// 开头, 带合法域名
  • 服务器必须可达 (浏览器打开 URL 能看到响应)
  • 防火墙 / 代理是否挡了外连? Mio Island 走的是 Socket.io (WebSocket)
  • 系统设置 → 通用 → Anthropic API 代理 字段影响不到 CodeLight 同步, 但你的系统代理会 (CodeLight 走 URLSession 默认用系统代理)
二维码出不来, 只显示转圈

说明服务器连接没成功。通常原因:

  • 服务器挂了 / DNS 错了 — 浏览器打开那个 URL 确认
  • 证书不被信任 (自签证书) — 导入到 macOS 钥匙串
  • 服务器 API 版本不匹配 — 确认 MioServer 是最新版
iPhone 配对成功了, 但状态不同步

99% 是 hook 没装或没开:

  • 系统设置 → 通用 → 钩子 开关打开
  • 检查文件存在: ls ~/.claude/hooks/codeisland-state.py
  • 检查 Claude Code 配置: ~/.claude/settings.json 里的 hooks 字段应该包含多个事件, 每个都指向 codeisland-state.py
  • 重启 Claude Code 会话 (老会话的 hook 是启动时加载的, 改了配置要重开)
一台 Mac 能配几台 iPhone? 一台 iPhone 能配几台 Mac?

都是多对多:

  • 同一个 6 位配对码可以被多台 iPhone 扫 (出差时用工作机 + 私人机都配)
  • 同一台 iPhone 可以连多台 Mac (办公 Mac + 家里 Mac) — 在 CodeLight 里切换
  • 服务器同时在线设备数取决于 MioServer 的并发限制 (默认单账户 1 台同时连)
服务器会看到我的代码 / 消息吗?

不会。通信协议是端到端加密 (Ed25519 签名 + 对称加密), 服务器只做转发, 不持久化内容。你如果不信, 可以自己看 MioServer 源码。

想完全卸载 / 停用 Mio Island 的钩子

系统设置 → 通用 → 钩子 关掉即可. 它会:

  • 从 ~/.claude/settings.json 里删除 Mio Island 的 hook entries (其他工具的 hooks 不动)
  • 删除 ~/.claude/hooks/codeisland-state.py
  • 刘海仍然可用, 但状态同步停了 (显示的是上一次快照)
